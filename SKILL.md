# 刷酒店攻略及体验内容生成器 (Hotel Guides Content Generator)

刷酒店攻略及体验内容生成工具，帮助创作者快速产出**图文并茂的 HTML 酒店攻略**及 Markdown 版本。

## 功能特性

- **智能标题生成** - 概述式、痛点式、悬念式、个性化等标题
- **正文结构化** - 概览+客房+餐饮+设施+难忘瞬间
- **标签自动匹配** - 精准标签组合，提升曝光率
- **照片智能识别** - 自动分析用户提供的照片，生成精确描述
- **内容自动填充** - 用照片识别内容替换模板占位描述
- **🆕 HTML 图文输出** - 生成可直接浏览器预览/分享的精美 HTML 文件（含图片嵌入）
- **🆕 双格式输出** - 同时生成 .md 和 .html 两个版本

## 输出规范

### 必须同时输出两种格式

每次执行任务时，**必须同时生成以下文件**：

| 文件 | 说明 |
|------|------|
| `[酒店名]酒店攻略.md` | Markdown 纯文本版（便于编辑） |
| `[酒店名]酒店攻略.html` | HTML 图文版（含图片嵌入，可直接预览/分享） |
| `images/` 目录 | 照片副本（相对路径引用） |

### HTML 输出标准

#### 视觉设计要求

- **配色方案**：深色主题 + 金色点缀（契合高端酒店调性）
  - 主色：`#1a1a2e`（深蓝黑）
  - 点缀色：`#c9a96e` / `#B8860B`（金色系）
  - 背景：`#f5f0eb`（暖白）
- **Hero 区域**：渐变背景 + 半透明封面图 + 入住信息 badge
- **图片展示**：卡片式布局，带圆角阴影和悬停动效
- **评分卡片**：醒目的数字评分 + 星级显示
- **标签墙**：圆角 tag 标签云
- **响应式设计**：适配手机端浏览

#### 图片处理规则

```
1. 将用户提供的照片复制到工作区的 images/ 子目录
2. HTML中使用相对路径引用：<img src="images/[文件名]">
3. 每张图片包裹在 .img-card 容器中：
   <div class="img-card">
     <img src="images/xxx.JPEG" alt="描述文字">
     <div class="img-caption">📍 简短说明</div>
   </div>
4. 图片按内容类型插入对应章节（见下方映射表）
```

### ✅ 正确方式：提供文件夹路径（推荐）

```
照片在 D:\我的酒店照片\ ，帮我生成成都华尔道夫酒店的攻略
```

**操作步骤：**
1. 将酒店照片放入一个文件夹（如 `D:\华尔道夫照片`）
2. 告诉 AI："照片在 **[文件夹路径]**，帮我生成 **[酒店名]** 攻略"
3. AI 会自动：
   - 扫描该文件夹中的所有图片文件
   - 根据文件名识别照片类型（外观/大堂/客房等）

   - 生成 `.md` + `.html` 双版本
   - 在浏览器中自动打开预览

**支持的文件夹格式：**
- 本地绝对路径：`D:\照片\`、`C:\Users\xxx\Desktop\`
- 路径中如有空格或中文，自动处理，无需引号

## 照片识别流程

### 当用户提供照片时

1. **接收照片文件**
   - 用户可通过 `@image:文件名` 引用照片
   - 或提供照片目录路径
   - 支持的文件格式：JPEG, PNG, JPG, WEBP

2. **复制照片到工作区**
   ```
   创建 images/ 子目录 → 复制所有照片到此目录
   ```

3. **分析照片内容**（多模态模型）
   - 识别酒店类型（大堂/客房/外观/餐厅等）
   - 提取视觉细节（装修风格、色调、布局、陈设）
   - 捕捉独特元素（景观、特色设施、设计亮点）

4. **生成照片描述**
   - 生成适合发布的图片描述文案
   - 与模板对应章节自动关联

5. **双格式输出**
   - **Markdown 版**：保留 `![描述](路径)` 格式
   - **HTML 版**：使用 `<div class="img-card">` 嵌入图片

## 最佳实践

### 内容安全准则

1. **避免敏感词**
   - 不要用："最"、"第一"、"保证"
   - 避免：价格（可写"性价比高"）、人身攻击、政治倾向

2. **真人感优先**
   - AI生成 → 人工微调 → 发布
   - 保留个人语言习惯
   - 照片描述应贴合实际视觉感受

3. **照片描述规范**
   - 客观描述为主 + 主观感受点缀
   - 长度控制在50-100字/张照片
   - 避免过度修辞，保持真实感

### 照片质量要求

- 建议分辨率：1080p以上
- 推荐角度：正面、45度角
- 光线：自然光或室内柔光
- 文件大小：不超过10MB

### HTML 输出检查清单

- [ ] 所有图片均通过相对路径 `images/xxx` 引用
- [ ] 每张图片都包裹在 `.img-card` 容器中并配有 `.img-caption`
- [ ] Hero 区域包含酒店中英文名 + 入住信息 badge
- [ ] 六大章节完整（概览/客房/餐饮/设施/难忘瞬间/总结）
- [ ] 评分卡片 + 推荐理由网格 + 建议 + 注意事项
- [ ] 标签墙包含 12-18 个精准标签
- [ ] CSS 包含响应式媒体查询 `@media (max-width: 600px)`
- [ ] 页脚包含免责声明
